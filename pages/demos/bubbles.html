CSS fit-content

Uses width: fit-content; max-width: 80%. The browser wraps the text, then sizes the bubble to the longest wrapped line. Shorter lines leave empty bubble area behind.

Wasted pixels: 0

Pretext shrinkwrap

Binary-searches the tightest width that still keeps the same line count. Same wrap, less wasted area, and no DOM text measurement in the resize path.

Wasted pixels: 0

Why can't CSS do this?

CSS can give you fit-content, which means “match the widest wrapped line.” It cannot ask for “the narrowest width that still gives me these same lines.” That requires checking multiple widths and keeping the line count constant. Pretext can do that from cached text metrics with pure arithmetic, so the resize path stays fast and DOM-free.
